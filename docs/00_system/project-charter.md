# Project Charter

## Status

Draft. Product direction is not approved yet.

## Purpose

DCOP starts with an operating harness before application code. The charter will define what the product is, who it serves, what success means, and which constraints must guide technical choices.

## Current Bootstrap Commitments

- No app stack, database, auth provider, payment provider, or deployment platform is selected yet.
- Product decisions must be recorded before implementation.
- Architecture or API decisions require ADR records.
- Risky changes require human approval.

## Required Before Implementation

- Product problem statement.
- Primary users and use cases.
- MVP success criteria.
- Out-of-scope items.
- Operational constraints.
